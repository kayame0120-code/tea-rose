## B. デザイントークン照合表（全マイルストーン共通）

Codexが実装したCSSが、下記の値と**完全一致**しているかをClaudeが照合する。独自の色値・余白を入れていたら不合格。

### B-1. カラートークン（モック `:root`・仕様書§25.1）
| トークン | 色値 | 主な用途 |
|---|---|---|
| `--cream` | `#FAF6F2` | 基本背景 |
| `--powder` | `#F4E7E4` | 淡い面・アクセント |
| `--rose` | `#D8A5A2` | 主要ボタン地色・アクセント枠 |
| `--beige` | `#EBE1D5` | 中間トーン面・区切り |
| `--tea` | `#4A3A32` | 主要文字色・強調枠・選択マーク |
| `--sage` | `#A4B6A0` | フォーカスリング |
| `--night` | `#2A211E` | ビューア・編集の暗背景 |

- ページ最外周の地色は `#ddd5cf`（トークン外の直値）。

### B-2. 文字色パレット14色（仕様書§8.6・M2で使用）

| ボタン | クラス | 地色 |

## 検品の進め方（監督AI=Claude）
1. Codex記入済みのA表を受け取る。各YESの根拠（ファイル・行）を実際に開く。
2. B表・C表と照合。色・文言の不一致は不合格。
3. 合否表（合格/修正/却下）を出す。NGは「どの正本のどこに反するか」を明記。
4. 根拠を見ていない項目は「未確認」と書き、YESにしない。
5. 合否は出すが確定はしない。進行宣言はあやめのタップのみ。
