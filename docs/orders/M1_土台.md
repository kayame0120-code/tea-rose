## このマイルストーンの範囲

写真が「取り込み → 変換 → 保存 → 一覧表示」で存在するまでを作る。ここが土台で、これが無いと編集もアルバムも試せない。

**含む：**
1. 写真選択画面（カメラカード・端末写真・デモ写真）
2. 取り込み時のHEIC/画像変換（canvas → JPEG）
3. IndexedDBへの保存
4. ストーリーホーム（日付別一覧・空表示・使用中バッジ）
5. 下部ナビ（ストーリー／アルバム／設定）※このM1ではストーリーのみ機能、他2つは画面枠だけでよい

**含まない（後続M）：** 写真編集＝M2 ／ アルバム＝M3 ／ ビューア・設定＝M4。

## 参照する確定仕様（この範囲に関係する箇所だけ）

### 画面：写真選択（仕様書§7）
- 見出し `写真をえらぶ`。上部に「端末の写真」行（`複数えらべます` ／ ボタン `写真をひらく`）。
- 選択グリッドは3列。**先頭セルに固定でカメラカード**（§7.2）。破線ローズ枠・左上カメラバッジ・中央にカメラアイコン＋文言 `その場でとる`。タップで `capture="environment"` のカメラ起動。
- ヘッダー右端は**空**。カメラ起動ボタンを置かない（旧⚡アイコンは撤去済み）。
- 端末写真：`input type="file"` `accept="image/*"` 複数選択可。
- デモ写真：初期13枚を選択グリッドにも表示（§7.5）。タップで複数選択、選択中は枠線＋選択順。
- 下部：`やめる` ／ `N枚を次へ`（未選択時は `次へ`・無効）。`やめる` で取り込んだ一時写真ごと破棄（§7.6）。
- カメラ結果トースト：成功 `写真を撮影しました` ／ キャンセル `撮影はキャンセルされました`（§7.4）。

### HEIC/画像変換（技術決定書 論点1＝方式B）
- 取り込み時に `createImageBitmap` で読み、canvasへ描き直し、`toDataURL("image/jpeg", 0.72)` でJPEG化。長辺1280pxへ縮小。
- **変換に失敗した写真は保存しない。** 失敗を知らせてスキップする（モックの「元ファイルをそのまま保存」フォールバックは採用しない）。
- 失敗時トーストの文言は**仕様書§17が持つ**。この発注書では文言を創作しない。§17に該当文言が無ければ実装を止めて質問する。

### 保存方式（技術決定書 論点2＝方式B）
- 画像実データ・photos/albumsのメタは **IndexedDB** に保存（同一端末内のみ・外部送信なし）。
- 保存失敗（容量超過等）時は仕様書§3の文言 `保存容量がいっぱいです。写真を減らしてください` のトースト。
- データモデルは仕様書§16の `Photo` / `Album` / `Overlay` / `Data` 型のまま。キー名・型名は変更しない（旧名可）。

### 画面：ストーリーホーム（仕様書§6）
- ヘッダー：見出し `ストーリー` ／ 英字補助 `story`。
- 写真登録ボタン：文言 `写真をとって あげる`（**スペース含む・変更禁止**）。地色 `--rose`／枠なし。押すと写真選択へ。
- 説明文（2行・§6.3のとおり一字一句）。
- 日付別一覧：表示日 `きょう` `きのう` `7月16日` `7月14日` `7月10日`。日付ごとに枚数表示。写真は縦長3:4・基本4列。
- 使用中バッジ：アルバムで使用中の写真にアルバム名バッジ最大2件。6文字超は `アルバム` と省略（§6.4）。※M1時点でアルバムは無いのでバッジは出ないが、表示ロジックは§6.4どおり実装しておく。
- きょうが0枚でも欄を表示。空表示文 `きょうはまだあげていません。` ＋ `「写真をとって あげる」から`。
- 写真タップで写真編集を開く導線＝M2で有効化。**M1では写真編集画面が未実装のため、タップ時は「開発予定」ではなく"M2で接続"のダミー遷移でよいか要確認**（下記・質問候補1）。

### 下部ナビ（仕様書§4）
- 主要画面でストーリー／アルバム／設定の3項目を下部固定。写真選択・写真編集では非表示。
- M1ではストーリーのみ中身あり。アルバム・設定はタップで空画面枠でよい（中身はM3・M4）。

### 初期データ（仕様書§15）
- 初期写真13枚（カテゴリは§15.1）・初期アルバム4件（§15.2）。M1では写真13枚が日付別一覧に正しく出ることまでを対象。
